Especificidad.

Ideas principales:

  1. Elementos inline (Especificidad: 1000)
  2. Selectores de ID (Especificidad: 0100)
  3. Selectores de clase, atributos y pseudoclases (Especificidad: 0010)
    • Clases (.Clase)
    • Atributos ([type="text"])
    • Pseudoclases (:hover, :nth-child)
  4. Selectores de elementos y pseudoelementos (Especificidad: 0001)
    • Elementos HTML (div, p, h1)
    • Pseudoelementos (::before, ::after)
  5. Universal, combinadores y no especificados (Especificidad: 0000)
    • Selector universal (*)
    • Combinadores (>, +, ~)

Reglas adicionales

  • último definido
  • !important

Apuntes de clase:

Especificidad: Es el medio por el cual los navegadores deciden que valores de CSS son más relevantes para aplicar.

Jerarquía de Especificidad en CSS:

  1. Elementos inline (Especificidad: 1000)
    • Son los estilos definidos directamente en el atributo style de un elemento.
  2. Selectores de ID (Especificidad: 0100)
    • Usan el atributo id del elemento.
  3. Selectores de clase, atributos y pseudoclases (Especificidad: 0010)
    • Incluyen:
      • Clases (.Clase)
      • Atributos ([type="text"])
      • Pseudoclases (:hover, :nth-child)
  4. Selectores de elementos y pseudoelementos (Especificidad: 0001)
    • Incluyen:
      • Elementos HTML (div, p, h1)
      • Pseudoelementos (::before, ::after)
  5. Universal, combinadores y no especificados (Especificidad: 0000)
    • Incluyen:
      • Selector universal (*)
      • Combinadores (>, +, ~)

Reglas adicionales

  • Si dos estilos tienen la misma especificidad, el último definido en el código tendrá prioridad.
  • El uso de !important anula todas las reglas de especificidad.

Resumen de la clase:

Especificidad en CSS determina qué reglas de estilo se aplican cuando hay conflictos. Los estilos inline tienen la mayor especificidad (1000), seguidos por los selectores de ID (0100), clases, atributos y pseudoclases (0010), elementos y pseudoelementos (0001), y los selectores universales y combinadores (0000). Si dos reglas tienen la misma especificidad, la última definida tendrá prioridad, y el uso de !important anula todas las reglas de especificidad.
